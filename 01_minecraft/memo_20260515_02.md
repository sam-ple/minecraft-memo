## Ping の意味

* 低いほど快適
* 単位は ms（ミリ秒）

目安：

| Ping      | 状態    |
| --------- | ----- |
| 0〜30ms    | 超快適   |
| 30〜60ms   | 快適    |
| 60〜100ms  | 普通    |
| 100〜150ms | 少しラグ  |
| 150ms以上   | 結構ラグい |

22ms はかなり良い数値です。

TAB はかなり高機能なので、少人数サーバーでも設定すると快適さがかなり上がります。

おすすめ設定を挙げると：

---

## 1. Ping色分け（超おすすめ）

Pingが見やすくなります。

`plugins/TAB/config.yml`

あたりにある：

```yaml id="lymx6g"
ping-thresholds:
```

を調整。

例：

```yaml id="92z5h7"
ping-thresholds:
  50: "&a"
  100: "&e"
  200: "&c"
```

意味：

* 50ms以下 → 緑
* 100ms以下 → 黄
* 200ms以上 → 赤

---

## 2. TAB一覧にTPS表示

サーバーの重さ確認に便利。

```yaml id="6dbfyz"
header-footer:
  enabled: true
```

のところを編集。

例：

```yaml id="m4n32v"
header:
- "&a&lMy Server"
- "&7TPS: %tps%"
- "&7Online: %online%"
```

すると Tab 上部に：

* TPS
* 接続人数

が出ます。

---

## 3. プレイヤー名の右に Ping 表示

かなり便利。

```yaml id="e2n9y9"
layout:
  enabled: true
```

や

```yaml id="bch1n3"
playerlist-objective
```

周辺を確認。

設定例：

```yaml id="13m2ga"
playerlist-objective:
  enabled: true
  value: "%ping%"
```

すると一覧に ms が表示されます。

---

## 4. 自分のサーバーなら TPS を最重要視

Pingより大事なのは TPS。

Paper なら：

```text id="bjlwm5"
/tps
```

で確認できます。

理想：

```text id="q6u2j7"
20.0
```

18以下になると重さを感じ始めます。

---

## 5. spark と組み合わせ最強

あなた既に：

spark

入れてるのでかなり良いです。

重くなったら：

```text id="f0z33o"
/spark tps
```

```text id="1g0n6m"
/spark profiler start
```

で原因特定できます。


保存後：


するとダブルクリック起動できます。
