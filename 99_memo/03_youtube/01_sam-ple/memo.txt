This is a remake of a beginner-friendly setup guide for Mine script, the mod that allows you to use Python in Minecraft, using the Modrinth app.
I’m demonstrating on a Mac with Modrinth, but the process is basically the same on Windows, Linux, the default Minecraft Launcher, or the CurseForge Launcher.
Using Python with Mine script is simple:
all you need to do is install the required mods, edit the config.txt to set your Python path, and then run your program.
<break time="2s"/>
Feel free to skip any parts that aren’t necessary for you.
<break time="5s"/>
First, download the Modrinth app and log in.
Then, create a new instance and install the required mods.
In this video, I’m using Java Edition 1.21.8 with Fabric.
For mods, Mine script alone is usually enough, but if you’re using Fabric, you also need the Fabric API.
Other mods are optional?feel free to add them as you like.
<break time="5s"/>
Next, set up the Python path in the config.txt.
First, find where the config.txt file is located.
Then, check your Python path.
On Mac, you can find it by running which python3 in Terminal.
On Windows, use where python in Command Prompt.
Once you have the path, open config.txt with your favorite text editor and set the Python location.
Here, I’m using vscode.

Finally, test your setup by running a program.
Sample programs are available on the official Mine script website.
Place your .py file in the same folder as Mine script, and save it as something like test.py or sample.py.

Be careful?it uses a backslash \, not a forward slash /.

Beginners Guide 01: TNT Underfoot Script in mine script
 <break time="2s"/>
In this guide, we’ll create a script that automatically places TNT under the player’s feet when standing on certain blocks.
 <break time="4s"/>
Step 1: Goal
 We want to detect the block below the player, and if it’s dirt or grass, replace it with TNT.
 <break time="4s"/>
Step 2: Running Scripts
 Place your .py file in the mine script folder.
 For example, if you have trapfeet.py, run it in chat with \trapfeet.
 To stop, use \killjob {id} or \killjob -1 for all jobs.
 <break time="5s"/>
Step 3: Imports
 We need three modules:
 mine script for Minecraft functions,
 Time for delays,
 And Math for safe coordinates.
 <break time="4s"/>
Step 4: Get Player Position
 Use m.player_position() and apply math.floor() to get accurate integer coordinates.
 <break time="4s"/>
Step 5: Check Block Below
 With m.getblock(x, y-1, z), we can see what block is under the player.
 <break time="4s"/>
Step 6: Place TNT
 Use m.execute("setblock … minecraft:tnt") to spawn TNT beneath the player.
 <break time="4s"/>
Step 7: Add a Block Check
 Only place TNT if the block matches dirt or grass.
 We use any() so it works even if the block has extra states like [snowy=true].
 <break time="5s"/>
Step 8: Continuous Loop
 Repeat the check every 0.1 seconds with time.sleep(0.1).
 <break time="4s"/>
Step 9: Function
 Wrap everything into a function called place_tnt_underfoot() for reuse.
 <break time="4s"/>
Step 10: Execution
 Finally, call the function and print “TNT underfoot activated!”.
 Optionally, use if __name__ == "__main__": for a safe standalone version.
 <break time="5s"/>

 You now have a working TNT underfoot script in mine script.

My First Build
<break time="1.2s"/>

I started playing Minecraft in April 2025, but to be honest, I had never really played it properly.
I had never gone on adventures or built anything—neither in single-player nor multiplayer.
<break time="1.4s"/>

In order to learn how Minecraft actually works, I recently joined a brand-new multiplayer server
that had just launched in September 2025,
and that’s where I decided to try building for the first time.
<break time="1.4s"/>

I didn’t know how to make polished stone, glass, or even concrete blocks.
I was starting completely from scratch.
<break time="1.3s"/>

Still, in the end, I think I managed to shape something
fairly close to what I had imagined.
<break time="1.4s"/>

Since I had put so much effort into it,
I decided to document it this time.
<break time="1.6s"/>

What I built was a very simple, box-shaped building,
with a rooftop,
three floors above ground,
and two underground floors.
<break time="1.5s"/>

The exterior is made entirely of white concrete blocks and polished stone,
and I used plenty of glass panes to make the whole building feel bright and open.
<break time="1.2s"/>

I built stairs using slabs,
added a farm in the basement,
set up ladders to move quickly between floors,
made it possible to exit directly from underground,
and even kept axolotls around the house.
<break time="1.4s"/>

Thinking through all these details
and building them step by step
was genuinely a lot of fun.
<break time="1.5s"/>

There was so much to learn that I even made my own cheat sheet,
but taking all of that into account,
I’m really satisfied with how it turned out.
<break time="1.4s"/>

On this server, I use a custom HUD I made with the Mine Script mod,
along with scripts created by other players.
<break time="1.2s"/>

They turned out to be incredibly useful,
and I’d like to keep making more things like that.
<break time="1.4s"/>

As for mods, I had barely used anything other than Mine Script before.
However, after installing several mods recommended on this server’s Discord,
I was honestly surprised by how convenient they were.
<break time="1.4s"/>

This server also has resource worlds,
lockable doors and chests,
an economy system,
dungeons,
and unique enchanted books.
<break time="1.2s"/>

I can even check my own progress and the world map on the web,
and I found that really interesting.
<break time="1.4s"/>

Now that the building itself is mostly finished,
I’ve been spending my time doing things like raising sniffers.
<break time="1.4s"/>

Going forward, there are so many things I want to try—
going on adventures,
completing more advancements,
assigning professions to villagers,
and building various farms and traps.
<break time="1.3s"/>

It’s all very exciting.
<break time="1.5s"/>

Minecraft really lets you do so many things.
Once again,
it reminded me just how deep the game really is.
<break time="1.8s"/>

And by the way,
I upgraded to MVP+ on the Hypixel server pretty much right away…
but I’ve barely played there at all.
<break time="1.8s"/>

Thank you for watching.

What is Autorun?

Autorun lets you start a script
automatically when entering a world.

No manual command needed.

🎬 Slide 3
Autorun is Simple.

You only need to add one line
to your config.txt.

🎬 Slide 4
For a Specific World

In config.txt:

autorun[WORLDNAME] = script_name

Replace:
WORLDNAME → your world name
script_name → script file name (without .py or .pyj)
Optional → add arguments after the script name

🎬 Slide 5
For All Worlds

In config.txt:

autorun[*] = script_name

The script will start
in every world.
