# Pythonを書く上で知っておくと便利な知識・コツ・TIPS。

---

## ✅ **文法・記法**

* リスト、辞書、タプルの**末尾カンマ `,` は許容される**し、複数行のときはむしろ推奨（PEP 8）。
* `if x:` や `if not x:` などの**真偽値チェックは簡潔に書ける**（`x != None` より `x is not None` 推奨）。
* **`val in list`** や **`val in dict`** は高速かつ読みやすい条件判定。
* **リスト内包表記**（`[f(x) for x in items if x > 0]`）はシンプルかつ高速。
* `enumerate()` を使うと、forループでインデックスを同時に扱える。
* `zip()` を使うと2つのリストをペアにしてループできる。
* Python 3.8以降では**ウォルラス演算子 `:=`**（代入式）で代入と条件を同時に書ける。

---

## 📦 **インポート・設計**

* `from module import *` は**衝突の元＆非推奨**。何が入ってくるか明示しないと後で困る。
* **インポート順は以下の順で空行を入れる**のがPEP 8推奨：

  1. 標準ライブラリ
  2. サードパーティライブラリ（例：`numpy`, `requests`）
  3. 自作モジュール
* `__all__` をモジュールに定義すれば `import *` でインポートされるものを制御できる。

---

## 🧠 **map, filter, lambda など**

* `map(func, iterable)` や `filter(func, iterable)` のように**ラムダや関数と組み合わせて使える**。
* 文字列処理・数値変換なども `map(str, list)` や `map(int, lines)` のように使うと簡潔。
* `list(map(...))` にしないとイテレータのままになる点に注意。
* `lambda x: x + 1` など**簡単な関数はlambdaで使い捨て可能**（ただし複雑な処理には使わない）。

---

## 🧹 **クリーンコード系のTIPS**

* **関数は短く・名前は明確に**（処理の目的がわかるように）。
* `docstring` を `"""説明文"""` で関数やクラスに書いておくと、可読性と補完の質が上がる。
* **定数は全て大文字 + アンダースコア（例：MAX\_SIZE）**。
* **型ヒント**（`def func(a: int) -> str:`）は補完や自動チェックに便利。

---

## 🐞 **デバッグ・例外処理**

* `try: ... except:` は最小限で書かず、**具体的な例外を捕捉する**（例：`except ValueError:`）。
* `with open(...) as f:` のように **with構文（コンテキストマネージャ）** を使うと自動でクローズされて安全。
* `print()` より `logging` を使うと、ログレベルや出力先を管理しやすい。

---

## 🧪 **テスト・開発の工夫**

* `assert` は条件が成り立たないときに例外を出すので**簡易チェックに便利**（テストや開発中に）。
* `pytest` などの**テストフレームワークで自動化**しておくと後から安心。
* `venv` を使って仮想環境を作るのが基本。依存を切り分けて事故を防ぐ。

---

## 🧰 **便利な組み込み関数・構文**

* `any([...])` や `all([...])` でリスト全体の真偽チェックができる。
* `sorted(list, key=...)` でソートの基準を柔軟に指定できる。
* `set()` を使うと重複排除や集合演算（和・積・差）ができる。
* `dict.get(key, default)` を使うと**キーがない場合の安全な取得**ができる。

---

## ⚡ **条件分岐の代替方法**

* **辞書**や**ラムダ式**を使うと、`if-elif-else` よりも簡潔に処理を記述できる場合があります。
* Python 3.10以降の\*\*`match-case`\*\*構文は、複雑な条件分岐をより柔軟に扱える強力なツールです。
* **関数を使った方法**は、条件ごとに異なる処理が必要な場合に便利です。

---

## 🗂️ **辞書の活用・安全な取得TIPS**

* `dict.get(key)` を使えば、**存在しないキーでも例外を出さず安全に取得できる**。
* `.get()` は型が混在していても柔軟に扱える（例：`"1"` と `1` を区別できる）。
* 条件付き処理では `if dict.get(key):` と書くと**未定義キーを自動でスキップ**できる。
* `.get(key, default)` を使えば**存在しないときにデフォルト値を使える**。
* `dict[key]` アクセスはキーがなければ `KeyError` になる点に注意。
* 辞書に値を動的に追加・カウントしたいなら `collections.defaultdict` も便利。
